# 项目进度报告

> 生成时间：2026-04-21

## 一、当前结论

项目已经从“工程骨架 + 三个方法首版”推进到“可系统实验比较 + M5 原型验证”的阶段。当前可稳定运行的主线包括：

- M1 `HermiteCubicMethod`
- M2 `HermiteQuinticMethod`
- M3 `ChebyshevMethod`
- M5 `RegularizedIterationMethod` 最小原型
- Gamma 验证、收敛扫描、Chebyshev 稳定性扫描、能量主对比、延拓域实验、M5 正则扫描

近期新增了三个关键能力：

- 类型 A（变底数 tetration）在 `z=0` 和 `z=-1` 的极限求值支持
- 高阶导数约束 API 拆分为“真正线性约束”“估计值线性化约束”“非线性残差诊断”
- M5 最小原型：固定端点值，在 Chebyshev-Lobatto 节点上用 least-squares 后端软最小化曲率能量和端点导数传播残差

基于最新实验结果，短期优先级建议是：**继续打磨 M5，M4 继续暂缓**。M5 的 least-squares 版本已经能稳定扫描 A/B/C，并显式暴露“Gamma 误差最优”“scaled residual 最优”和“raw 高阶残差”之间的权衡；下一步应围绕 residual scaling 与多区间泛函方程残差继续设计实验。

导数约束 API 现在分为三类：

- `linear_derivative_constraints(max_order)`：只返回可直接进入线性约束 QP 的真实线性关系。
- `linearized_derivative_constraints(max_order, deriv_estimates)`：给非线性高阶约束提供基于估计导数的线性化/冻结版本。
- `derivative_constraint_residuals(max_order, left_derivatives, right_derivatives)`：用于评估端点导数传播的真实残差，供后续 M5 或非线性求解器使用。

## 五、已知问题与风险

### 5.1 Chebyshev 条件数仍高

当前 Chebyshev 使用系数空间、端点导数解析权重、约束行缩放和 Schur-complement 求解，已经比早期 nodal 微分矩阵版本稳定。但在 FactorialType 默认设置下，Schur 条件数仍约为 `6.98e6`，A/B 约为 `4e4`。

风险：继续提高 degree 或 constraint_order 时，误差可能改善但病态性也会迅速加重。

### 5.2 A/B 的高阶非线性约束不能直接塞进线性 QP

类型 A/B 的二阶及以上导数传播含非线性项。现在 API 已拆开，但默认 Chebyshev 只使用真实线性约束；如果开启 `use_linearized_constraints=True`，仍需要通过实验确认收敛性和稳健性。

### 5.3 单一区间能量最小化不等于全局自然性

能量实验显示 M1 在 A/B/C 上给出最低基底应变能，但 Gamma 精度并非最佳。这说明单一区间曲率能量只是“自然性”的一个局部指标，后续需要同时看：

- Gamma 或其他 ground truth 误差
- 端点导数传播残差
- 区间内部泛函方程残差
- 区间外延拓稳定性

### 5.4 类型 A 正向延拓很快溢出

`z=5` 已经出现 overflow。后续如果要研究更远正向延拓，需要考虑：

- log-domain 递推
- 溢出前停止与量级报告
- 多精度浮点

### 5.5 M5 仍然只是单区间端点残差原型

当前 M5 的 residual 项主要是端点导数传播残差，而不是完整的多区间内部泛函方程残差。因此它已经可以和 Chebyshev 的“硬约束 QP”做对照，但还不能完全代表实验方案中更一般的“泛函方程残差 + 平滑正则”方法。

### 5.6 pycache 跟踪已清理

已在独立提交 `57dd8b5 chore: stop tracking Python cache files` 中新增 `.gitignore` 并从索引移除历史 `__pycache__/*.pyc`。后续运行测试和编译不应再污染 Git 状态。

## 六、下一步建议

### 6.1 继续推进 M5 v2

建议围绕首轮扫描继续升级 M5：

- 增加多区间窗口，例如同时表示 `[n0, n0+2]`，让内部采样点真正评价 `f(z) - g(z, f(z-1))`。
- 继续比较 `absolute / relative / order_weighted` residual scaling，避免 scaled residual 好看但 raw 四阶残差失控。
- 评估 `least_squares` 的 `trf/dogbox` 与 `minimize` 后端在不同 residual scaling 下的稳定性。
- 将 `initial_method` 扩展为可选 Chebyshev 和上一次 M5 最优配置 warm start。

### 6.2 给 M5 补对照实验

建议新增或扩展：

- 在 `exp_regularized_iter.py` 中加入 M1/M2/M3 baseline 对照行。
- 把 `exp_extension.py` 纳入 M5，比较 M5 在 `z=0/-1/3/4/5` 的延拓稳定性。
- 对 FactorialType 同时看 Gamma 误差、端点残差、应变能，避免只按单一指标选模型。

### 6.3 M4 继续暂缓

当前还没有证据显示“单一区间全局基函数不足以表达局部结构”是主要瓶颈。相比之下，A/B 的高阶非线性残差与区间外稳定性问题更直接。因此 M4 可以等 M5 跑出结果后再决定。

### 6.4 Chebyshev 后续小改进

Chebyshev 不需要立刻大改，但建议保留两条轻量增强：

- 给 `autotune()` 增加是否启用 `use_linearized_constraints` 的候选项。
- 在实验表中长期保留 `constraint_mode`、`constraint_count`、`schur_condition_number`，防止“精度提升但条件数爆炸”的假进步。
