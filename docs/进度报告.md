# 项目进度报告

> 生成时间：2026-04-21

## 一、当前结论

项目已经从“工程骨架 + 三个方法首版”推进到“可系统实验比较”的阶段。当前可稳定运行的主线包括：

- M1 `HermiteCubicMethod`
- M2 `HermiteQuinticMethod`
- M3 `ChebyshevMethod`
- Gamma 验证、收敛扫描、Chebyshev 稳定性扫描、能量主对比、延拓域实验

本轮新增了两个关键能力：

- 类型 A（变底数 tetration）在 `z=0` 和 `z=-1` 的极限求值支持
- 高阶导数约束 API 拆分为“真正线性约束”“估计值线性化约束”“非线性残差诊断”

基于最新实验结果，短期优先级建议是：**先实现 M5 的小型原型，M4 继续暂缓**。原因是当前数据已经显示，全局单区间方法在端点约束和 Gamma 精度上可用，但对 A/B 的高阶非线性残差仍很大；M5 的“泛函方程残差 + 平滑正则”的目标函数更适合回答下一阶段问题。

| 文件 | 状态 | 当前说明 |

导数约束 API 现在分为三类：

- `linear_derivative_constraints(max_order)`：只返回可直接进入线性约束 QP 的真实线性关系。
- `linearized_derivative_constraints(max_order, deriv_estimates)`：给非线性高阶约束提供基于估计导数的线性化/冻结版本。
- `derivative_constraint_residuals(max_order, left_derivatives, right_derivatives)`：用于评估端点导数传播的真实残差，供后续 M5 或非线性求解器使用。

## 三、最新实验结果

### 3.1 Gamma 验证

运行命令：

```bash
conda run --no-capture-output -n generic python experiments/exp_gamma_validation.py
```

最新结果：

| 方法 | `z=0.5` 误差 | `z=1.5` 误差 | `z=2.5` 误差 |
|------|-------------:|-------------:|-------------:|
| M1 三次 Hermite | 6.630e-03 | 9.945e-03 | 2.486e-02 |
| M2 五次 Hermite | 3.994e-04 | 5.991e-04 | 1.498e-03 |
| M3 Chebyshev | 1.805e-04 | 2.707e-04 | 6.768e-04 |

结论：

- M2 相比 M1 稳定提升约一个数量级。
- 调整后的 Chebyshev 默认主线在 FactorialType 上优于 M2。
- Chebyshev 的 Schur 条件数仍显著高于 Hermite，需要继续监控稳定性。

|------|------:|-------:|

失败点符合预期：

- A 在 `z=-2` 遇到变底数反向递推奇点。

- A/B 的一阶约束残差接近机器精度。
- A/B 的三阶、四阶非线性残差较大。
- C 的 Chebyshev 因使用了 FactorialType 的高阶线性约束，1-4 阶端点残差约为 `1e-12` 量级。

这个结果支持下一步做 M5：当前方法可以满足端点低阶约束，但还没有系统最小化区间内泛函方程残差。

## 五、已知问题与风险

### 5.1 Chebyshev 条件数仍高

当前 Chebyshev 使用系数空间、端点导数解析权重、约束行缩放和 Schur-complement 求解，已经比早期 nodal 微分矩阵版本稳定。但在 FactorialType 默认设置下，Schur 条件数仍约为 `6.98e6`，A/B 约为 `4e4`。

风险：继续提高 degree 或 constraint_order 时，误差可能改善但病态性也会迅速加重。

### 5.2 A/B 的高阶非线性约束不能直接塞进线性 QP

类型 A/B 的二阶及以上导数传播含非线性项。现在 API 已拆开，但默认 Chebyshev 只使用真实线性约束；如果开启 `use_linearized_constraints=True`，仍需要通过实验确认收敛性和稳健性。

### 5.3 单一区间能量最小化不等于全局自然性

能量实验显示 M1 在 A/B/C 上给出最低基底应变能，但 Gamma 精度并非最佳。这说明单一区间曲率能量只是“自然性”的一个局部指标，后续需要同时看：

- Gamma 或其他 ground truth 误差
- 端点导数传播残差
- 区间内部泛函方程残差
- 区间外延拓稳定性

### 5.4 类型 A 正向延拓很快溢出

`z=5` 已经出现 overflow。后续如果要研究更远正向延拓，需要考虑：

- log-domain 递推
- 溢出前停止与量级报告
- 多精度浮点

### 5.5 当前目录不是 Git 仓库

本轮检查中 `git status` 提示当前目录不是 Git 仓库。因此目前只能依赖文件内容、测试和实验输出进行变更追踪。建议之后若要长期迭代，初始化或恢复 Git 管理。

## 六、下一步建议

### 6.1 优先做 M5 最小原型

建议先实现一个小而清晰的 `RegularizedIterMethod`：

- 在 `[n0, n0+1]` 取 Chebyshev-Lobatto 或均匀节点。
- 固定端点值 `f(n0), f(n0+1)`。
- 目标函数设为 `lambda_energy * J(f) + lambda_residual * R(f)`。
- `J(f)` 使用二阶差分或已有 Chebyshev/离散能量。
- `R(f)` 先定义为端点导数残差 + 若干采样点的递推一致性残差。
- 对 FactorialType 先跑通，并与 Gamma 对照。

这条路线最能直接回答：“只压低曲率是否不够？加入泛函残差后，A/B/C 的延拓会不会更稳定？”

### 6.2 给 M5 配套实验

新增建议脚本：

```text
experiments/exp_regularized_iter.py
```

扫描：

- `lambda_energy`
- `lambda_residual`
- 节点数 `N`
- 初值来源：M1 / M2 / Chebyshev

输出：

- Gamma 误差
- 应变能
- 端点残差
- 区间内残差
- 延拓域失败点

### 6.3 M4 继续暂缓

当前还没有证据显示“单一区间全局基函数不足以表达局部结构”是主要瓶颈。相比之下，A/B 的高阶非线性残差与区间外稳定性问题更直接。因此 M4 可以等 M5 跑出结果后再决定。

### 6.4 Chebyshev 后续小改进

Chebyshev 不需要立刻大改，但建议保留两条轻量增强：

- 给 `autotune()` 增加是否启用 `use_linearized_constraints` 的候选项。
- 在实验表中长期保留 `constraint_mode`、`constraint_count`、`schur_condition_number`，防止“精度提升但条件数爆炸”的假进步。
